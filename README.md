#三国版泡泡堂游戏说明
***写游戏过程中遇到的问题及解决方法***:  
           1.支持多个英雄人物，有用核自旋，并用“英雄名”为索引创建英雄动画。  
           2.场景间传参已达成人物以及地图的选取，重载场景使用create（）和createScene（）。  
           3.障碍物支持，瓦片地图障碍层的使用和检测。  
           4.爆炸波四个方向的不同延伸，爆炸波遇到障碍物的延伸停止，并判断此障碍物是否可炸，     
            单独创设爆炸波类，将爆炸波单位化设置。  
           5.人物与炸弹的碰撞检测以及人物与爆炸波的碰撞检测，可炸障碍物被炸除后三种道具的  
           等概率随机产生以及人物对道具的拾取，道具产生以及拾取的动态修改地图中各层级图快。  
           6.人物运动机制：基于update函数的高频率setPosition操作，数次修改以保证人物在障碍  
            物附近的移动的支持。  
           7.网络持续连接问题：采用static关键字声明定义的客户端类对象cClient，在场景切换时  
           传递指向该类对象的指针以让cClient在下一个场景中也能进行数据的传输。  
           8.联网对战（网络通信）：  
             使用boost.asio异步实现tcp通信  
             实现方法  
             选择角色场景：  
             客户端点击联网对战进入选择角色画面，进行登陆，服务端回显客户端id（playerX，X=1，2）  
             客户端接受id，并以此作为游戏id。  
             客户端决定选择角色，发送角色选择消息（6XplayerY）。  
             服务端持续接收角色选择消息，当决定人数等于游戏人数，服务端给每个参与者发送角  
             色资料和游戏开始指令。  
             游戏场景： 当客户端进行操作时，客户端发出消息给服务端。  
             服务端接受消息并回返给每一位游戏参与者。  
             游戏参与者处理接受的消息并在客户端上更新人物状态。  

***实现的功能***:  
***基础功能***：  
               1.支持地图绘制、人物绘制、水泡效果绘制等  
               2.支持鼠标和键盘操作交互  
               3.支持障碍物  
               4.支持泡泡的放置与爆炸  
               5.支持三种基本增强型道具（鞋子，泡泡，药水）

	           3.支持4种游戏模式，单机双人对战，单机单人迷宫模式，单机双人命运对决，联机双人对战

***游戏模式及玩法介绍***：  
***双人模式***：  
               玩家一：使用W，S，A，D控制方向移动，空格或J施放炸弹  
               玩家二：使用↑，↓，←，→控制方向移动，Enter施放炸弹  
               死亡一个玩家即游戏结束  
***单人迷宫模式***：  
               玩家使用W，S，A，D控制方向移动，空格或J施放炸弹  
               玩家到达指定的正确宝箱位置获得胜利  
***命运对决模式**：  
               玩家一：使用W，S，A，D控制方向移动，空格或J施放炸弹  
               玩家二：使用↑，↓，←，→控制方向移动，Enter施放炸弹  
               道具爆率为双倍爆率，炸弹威力每次会在1到两倍当前最大威力间随机出现，  
               死亡一个玩家即游戏结束  
***联机对战***：  
               玩家均使用W，S，A，D控制方向移动，空格或J施放炸弹，死亡一个玩家即游戏结束  

***游戏特色***:  
               整个游戏界面采取三国古朴风，游戏人物为赵云，周瑜，孙尚香，黄月英。  
               地图则有草原，冰雪，沙漠，城区4个不同的场景风格
